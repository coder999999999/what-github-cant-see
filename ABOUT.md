# About this project

**This is an art project. It is commentary, not fraud.**

If you found this repo because you were annoyed, confused, or suspicious
about the contribution graph attached to it — that is the entire point.
Please read this before reporting.

---

## What this is

"What GitHub Can't See" is a conceptual piece — and a small piece of
research — about how the software industry uses commit counts and
contribution graphs as a proxy for effort, skill, and legitimacy, and
how meaningless that proxy has become in an era where AI agents can
generate thousands of commits a day with a single command.

Part of the reason this repo exists is that I wanted to find out
whether it was even possible: whether one person, on a laptop, in an
afternoon, could convincingly fake the thing the industry treats as
evidence of work. The answer turned out to be yes, trivially, and
that finding is most of the point.

The repository contains no product, no library, no functional code, and
no attempt to deceive anyone into thinking it does. The commit history
is the artwork. The README is the artist statement. The contribution
graph is the canvas.

## What this is *not*

- **Not spam.** Nothing here is pushed to any package registry, no
  dependencies are claimed, no users are targeted.
- **Not malware.** The only script (`draw.py`) writes backdated commits
  to a local git repository. It does nothing else. Read it.
- **Not impersonation.** Every commit is authored by me, on my own
  account, in my own repo.
- **Not a attempt to game hiring.** The README explicitly tells any
  recruiter that these commits are fake. That is the piece.
- **Not an attack on GitHub.** GitHub's contribution graph is doing
  exactly what it was designed to do. The critique is aimed at how
  people *read* it, not at the platform.

## Why the contribution graph looks the way it does

The graph is filled with 99,999 backdated commits generated by a
script (`draw.py`) in under a minute. In 2024 and 2025 the lit squares
spell the word **CODER**. The point is that a contribution graph can
be arbitrarily shaped with trivial effort, and therefore tells you
almost nothing about the person attached to it.

## Reversibility

If this repository is deleted, the associated commits drop off the
contribution graph automatically within minutes to hours. Nothing here
is permanent. Nothing here leaves a footprint outside this repo.

## Lineage

Commit-graph art is not new. Projects like `gitfiti`, `github_painter`,
and others have existed for years. What makes this piece different is
not the mechanism — it's the framing: doing the thing the industry is
currently anxious about, as loudly as possible, and labeling it clearly
so the conversation can be about the framing rather than the trick.

1. **The volume of commits.** Yes — that is the critique.
2. **That they contain no real work.** Yes — that is the critique.
